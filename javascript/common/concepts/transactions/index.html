Transactions

A write transaction allows a device to perform multiple operations within a single database call. You can perform insert, update, remove or evict operations on multiple collections more efficiently using this method.

Not supported by JavaScript.
ditto.store.write { transaction in    let cars = transaction.scoped(toCollectionNamed: "cars")    let people = transaction.scoped(toCollectionNamed: "people")    let docId = "abc123"    try! people.upsert(["_id": docId, "name": "Susan"])    try! cars.upsert(["make": "Ford", "color": "red", "owner": docId])    try! cars.upsert(["make": "Toyota", "color": "black", "owner": docId])    people.findByID(docId).evict()}
NSArray *results = [store write:^(DITWriteTransaction *tx) {    DITScopedWriteTransaction *cars = tx[@"cars"];    DITScopedWriteTransaction *people = tx[@"people"];    DITDocumentID *docID = [[DITDocumentID alloc] initWithValue: @"abc123"];    [people upsert:@{@"_id": docID, @"name": @"Susan"} error:nil];    [cars upsert:@{@"make": @"Ford", @"color": @"black", @"owner": docID} error:nil];    [cars upsert:@{@"make": @"Toyota", @"color": @"red", @"owner": docID} error:nil];}];
SuppressUnusedWarning(results);
val results = ditto.store.write { transaction ->    val cars = transaction.scoped("cars")    val people = transaction.scoped("people")    val docId = "abc123"    people.upsert(mapOf("_id" to docId, "name" to "Susan"))    cars.upsert(mapOf("make" to "Hyundai", "color" to "red", "owner" to docId))    cars.upsert(mapOf("make" to "Jeep", "color" to "pink", "owner" to docId))    people.findByID(DittoDocumentID(docId)).evict()}
Not supported by Java.
Not supported by C#.
Not supported by C++.
Not supported by Rust.
Not supported by HTTP.
warning

You must use the scoped collection within a write transaction, as in the above example. For example, txn.scoped("collection_name").find(query).exec().

Using ditto.store.collection(...) will cause a panic at runtime.
